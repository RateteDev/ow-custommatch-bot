# 025-match-restart-confirm

## Context

同じ channel で `/match` を再実行した際、現状は「既に募集が開始されています」と
エフェメラルで通知するだけで、利用者は次に何をすべきか判断しづらいです。

既存募集を閉じて新しく始めるのか、そのままキャンセルするのかを、
エフェメラルの確認 UI で選べるようにします。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `internal/bot/bot.go` | `/match` 再実行時の確認 UI と操作ハンドラを追加 |
| `internal/bot/bot_test.go` | 確認 UI 表示、承認、キャンセルの回帰テストを追加 |

## 実施順序

1. 再実行時に確認 UI を返す分岐を追加
2. 「閉じて開始」「キャンセル」の component handler を追加
3. 発案者本人のみ操作できる制御を入れる
4. 各経路のテストを追加する

## 要件

### 要件1: 再実行時の確認メッセージ

- 同一 channel に open な募集がある場合は、単純エラーではなく確認 UI を返す
- 確認はエフェメラルで表示する

### 要件2: 募集の再開処理

- 「既存の募集を閉じて開始」を選んだ場合のみ、新しい募集を開始する
- 既存募集の状態更新と新規作成の整合を保つ

### 要件3: キャンセル処理

- 「キャンセル」を選んだ場合は何も変更しない

### 要件4: 権限制御

- 確認 UI は発案者本人のみが利用できる前提にし、想定外操作を拒否する

## 検証方法

- `go test ./internal/bot/...`
- 同じ channel で `/match` を再実行し、確認 UI が表示されることを手動確認
