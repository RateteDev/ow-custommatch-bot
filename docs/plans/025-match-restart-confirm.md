# 025-match-restart-confirm

## Context

同じ channel で `/match` を再実行した際、現状は「既に募集が開始されています」と
エフェメラルで通知するだけで、利用者は次に何をすべきか判断しづらいです。

ユーザーコメントでも、
「既存のものを閉じるか・キャンセルするかのエフェメラルメッセージを出す想定ではないか」
という指摘が出ています。

既存募集を閉じて新しく始めるのか、そのままキャンセルするのかを、
エフェメラルの確認 UI で選べるようにします。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `internal/bot/bot.go` | `/match` 再実行時の確認 UI と操作ハンドラを追加 |
| `internal/bot/bot_test.go` | 確認 UI 表示、承認、キャンセル、想定外操作の回帰テストを追加 |

## 実施順序

1. 再実行時に確認 UI を返す分岐を追加
2. 「閉じて開始」「キャンセル」の操作を処理する handler を追加
3. 想定外ユーザーや古い UI からの操作を安全側で拒否する
4. 表示・承認・キャンセル・競合時のテストを追加する

## 要件

### 要件1: `/match` 再実行時に確認 UI を表示する

- 同じ channel に open な募集がある場合は、単純エラーで終了しない
- 利用者に次の 2 択を提示する
  - 既存の募集を閉じて開始
  - キャンセル
- 確認 UI はエフェメラルで表示する

#### 表示メッセージの例

```text
このチャンネルでは既に募集が開始されています。

現在の募集を閉じて、新しく募集を開始しますか？
現在の参加者一覧は引き継がれません。

[既存の募集を閉じて開始] [キャンセル]
```

#### 完了条件

- `/match` 再実行時に、利用者が次の行動をその場で選べる
- 「既に募集されています」で終わらない

### 要件2: 「既存の募集を閉じて開始」を選んだときの挙動を明確にする

- 既存募集は open 状態ではなくなる
- 同じ channel に対して新しい募集を 1 件だけ開始する
- 新しい募集メッセージは通常の `/match` 開始時と同じ見た目にする

#### 期待結果

- 古い募集は entry / assign / cancel の対象外になる
- 新しい募集だけが有効になる
- 同じ channel に open な募集が 2 件並立しない

### 要件3: 「キャンセル」を選んだときは何も変更しない

- 既存募集の状態を変えない
- 新しい募集も作らない
- 誤って開いた確認 UI から安全に戻れる

#### 期待結果

- 既存の募集メッセージ、参加者一覧、open 状態がそのまま残る

### 要件4: 想定外操作に強いこと

- 確認 UI を開いた本人だけが、その UI の決定操作を完了できるようにする
- UI 表示から決定までの間に既存募集の状態が変わっていても、壊れた状態を作らない

#### 確認したいケース

- A さんが確認 UI を出したあと、別ユーザーが同じ channel で操作しても不整合にならない
- UI 表示後に既存募集がすでに閉じられていた場合でも、二重開始や重複 open が起きない

### 要件5: 回帰テストを追加する

- 確認 UI が表示されること
- 「閉じて開始」で新規募集に切り替わること
- 「キャンセル」で何も変わらないこと
- 想定外操作が拒否されること
- 既存の別 channel 独立性や二重開始防止を壊していないこと

## 検証方法

- `go test ./internal/bot/...`
- 同じ channel で `/match` を再実行し、確認 UI が表示されることを手動確認
- 「閉じて開始」で新しい募集に切り替わることを手動確認
- 「キャンセル」で既存募集が維持されることを手動確認
