# 034-startup-mode-confirmation-copy

## Context

起動方法の選択肢はすでに `通常運用` / `動作確認用` の日本語に揃っていますが、
起動確定メッセージだけは `PROD 通常運用で起動します。` /
`TEST 動作確認用で起動します。` のまま残っています。

`PROD` と `TEST` は開発者には通じても、利用者視点では
起動メニューの文言と一致せず、追加の読み替えが必要になります。

起動前後で同じ言葉だけが見える状態に揃え、
利用者が迷わず現在の起動モードを理解できる表示へ統一します。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `cmd/ow-custommatch-bot/main.go` | 起動確定メッセージから `PROD` / `TEST` を除去し、起動メニューの語彙へ統一 |
| `cmd/ow-custommatch-bot/main_test.go` | 起動確定メッセージの期待値を更新 |
| `docs/CLI表示仕様.md` | 起動確定メッセージの仕様を更新 |

## 実施順序

1. 先に起動確定メッセージのテスト期待値を更新する
2. `startupModeConfirmationMessage` の返却文言を変更する
3. 仕様書の表示例を新文言へ更新する

## 要件

### 要件1: 通常運用の確定文言から `PROD` を削除する

- 通常運用を選んだ直後の表示は `通常運用で起動します。` とする
- `PROD 通常運用で起動します。` は表示しない
- 起動メニューの `[1] 通常運用` と同じ語彙を使う

### 要件2: 動作確認用の確定文言から `TEST` を削除する

- 動作確認用を選んだ直後の表示は `動作確認用で起動します。` とする
- `TEST 動作確認用で起動します。` は表示しない
- 起動メニューの `[2] 動作確認用` と同じ語彙を使う

### 要件3: 色分けは維持し、意味は文言だけで伝わる状態にする

- 通常運用時は現行どおり緑系の表示を維持してよい
- 動作確認用時は現行どおり黄系の表示を維持してよい
- ANSI 色が無効な環境でも、文言だけでモードが判別できること

### 要件4: 表示回帰をテストで固定する

- `startupModeConfirmationMessage(false)` は `通常運用で起動します。` を返すこと
- `startupModeConfirmationMessage(true)` は `動作確認用で起動します。` を返すこと
- 既存テスト内に `PROD ` と `TEST ` の期待値を残さないこと

## 検証方法

- `go test ./cmd/ow-custommatch-bot/...`
- 対話起動で `[1]` と `[2]` を選び、起動確定文言がそれぞれ新文言になることを確認
- `docs/CLI表示仕様.md` の起動確定メッセージ節が実装と一致することを確認
