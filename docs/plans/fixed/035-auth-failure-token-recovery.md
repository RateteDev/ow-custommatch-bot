# 035-auth-failure-token-recovery

## Context

現在は保存済みトークンで起動したあとに Discord 側から
`websocket: close 4004: Authentication failed.` が返ると、
`実行中にエラーが発生しました` を表示してそのまま終了します。

この失敗は利用者が自力で直せる可能性が高い一方で、
現状の画面には「どこでトークンを確認・再発行するか」
「次にこのアプリで何をすればよいか」が不足しています。

認証失敗時は単なる異常終了ではなく、
Discord Developer Portal への案内とトークン再入力導線をその場で提示し、
保存済みトークンの更新まで起動フロー内で完結できるようにします。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `cmd/ow-custommatch-bot/main.go` | `4004 Authentication failed` を認証失敗として判定し、案内表示とトークン再入力フローを追加 |
| `cmd/ow-custommatch-bot/main_test.go` | 認証失敗時の案内文言、トークン再保存、再試行動作のテストを追加 |
| `docs/CLI表示仕様.md` | 認証失敗時の案内と復旧フローを追記 |
| `assets/windows/使い方.html` | トークン取得先と再設定手順を利用者向けに追記 |

## 実施順序

1. 先に `4004 Authentication failed` を認証失敗として扱う判定テストを追加する
2. 認証失敗時の案内文言と再設定フローのテストを追加する
3. Bot 実行失敗時に認証失敗だけを分岐し、トークン上書きへ遷移させる
4. 利用者向けドキュメントを実装どおり更新する

## 要件

### 要件1: `4004 Authentication failed` を通常エラーと分けて扱う

- `b.Run(botToken)` が返すエラー文字列に `4004` と `Authentication failed` が含まれる場合は、認証失敗として扱う
- 認証失敗時は `実行中にエラーが発生しました: websocket: close 4004: Authentication failed.` のみを出して終了しない
- `4004` を含まない一般的な実行エラーは現行どおり終了する

### 要件2: 利用者向け案内に復旧先 URL を明示する

- 認証失敗時は `https://discord.com/developers/applications` を画面に表示する
- 案内文には「Discord Developer Portal で BOT TOKEN を再発行または確認してください」を明記する


### 要件3: 認証失敗後はトークン入力モードへ移行する

- 認証失敗を検知したら、その場で保存済みトークンの上書きフローへ入る
- 上書き時は既存の `promptBotToken` と `saveTokenToStoreFn` を使い、保存先は現行どおり Credential Manager とする
- 新しいトークンの保存に成功したら、同じプロセス内で Bot 起動を 1 回だけ再試行する
- 再試行後に接続成功した場合は通常どおり `ready()` を表示する

### 要件4: 再設定を中断した場合の終了理由を明確にする

- 認証失敗後のトークン入力が空だった場合は、`トークンが入力されませんでした。` を含む案内で終了する
- トークン保存に失敗した場合は、`トークンの保存に失敗しました` を含む案内で終了する
- 認証失敗後の再試行でも再度 `4004 Authentication failed` になった場合は、再試行失敗を明示して終了する
- 認証失敗からの自動再試行回数は 1 回に固定し、無限ループにしない

### 要件5: テストで復旧フロー全体を固定する

- 認証失敗時に Developer Portal URL を含む案内が表示されること
- 認証失敗後に入力したトークンが保存されること
- 新トークンで Bot 実行が 1 回だけ再試行されること
- 認証失敗以外の実行エラーでは再設定フローに入らないこと

## 検証方法

- `go test ./cmd/ow-custommatch-bot/...`
- 保存済み無効トークンで起動し、Developer Portal URL 表示後に新トークン入力へ進むことを手動確認
- 新トークン入力後に接続成功すればそのまま利用継続できることを手動確認
- 認証失敗以外の実行エラーでは従来どおり終了することを手動確認
