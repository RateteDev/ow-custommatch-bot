# 026-token-store-management

## Context

`BOT_TOKEN` は Windows Credential Manager に保存されるようになりましたが、
利用者向けに「確認」「上書き」「削除」をどう行うかの導線がありません。

現状だと、初回設定までは分かっても、
「別のトークンに切り替えたい」
「保存済みトークンを消して最初からやり直したい」
「今どこに保存されているのか知りたい」
といった運用時の疑問に答えられません。

初回設定だけでなく、トークン更新やトラブル時の再設定まで自己解決できるよう、
アプリ内導線と案内文書を整備します。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `cmd/ow-custommatch-bot/main.go` | トークン再設定や削除に関する CLI 導線を追加 |
| `cmd/ow-custommatch-bot/token_windows.go` | 必要に応じて削除用の資格情報ストア操作を追加 |
| `cmd/ow-custommatch-bot/main_test.go` | 追加導線のテストを追加 |
| `README.md` | TOKEN の保存場所、上書き、削除方法を追記 |
| `assets/windows/使い方.html` | 利用者向け手順を追記 |

## 実施順序

1. 利用者に提供する操作を決める
2. 資格情報ストア操作を実装する
3. CLI / エラーメッセージ / ヘルプの導線を整える
4. README と使い方ガイドを更新する

## 要件

### 要件1: 利用者ができる操作を明確にする

- 少なくとも次の 3 操作について、利用者が方法を理解できる状態にする
  - 保存済みトークンで通常起動する
  - 新しいトークンに上書きする
  - 保存済みトークンを削除する
- 「確認」はトークン値そのものの表示ではなく、
  保存先と現在の運用状態が理解できることを指す

#### 解消したい疑問

- `BOT_TOKEN` はどこに保存されているのか
- 新しいトークンに変えたいときは何をすればよいのか
- 保存を消して最初からやり直すにはどうするのか

### 要件2: 上書き導線を用意する

- 既存のトークンを、新しいトークンで更新できる手段を用意する
- 利用者が「次回以降は新しいトークンで起動する」と理解できる案内にする

#### 操作例

```text
ow-custommatch-bot.exe --reset-token
  -> 新しい BOT_TOKEN を入力してください
  -> 保存済みトークンを更新しました
```

上記は一例です。実際の導線は CLI オプションでも起動メニューでも構いませんが、
README と `--help` から辿れる必要があります。

### 要件3: 削除導線を用意する

- 資格情報ストア上の `BOT_TOKEN` を削除できる手段を用意する
- 削除後は次回起動時に初回設定と同じ扱いになる

#### 期待結果

- 削除後に通常起動すると、保存済みトークンが見つからない扱いになる
- 再入力すれば復旧できる

### 要件4: 利用者向け案内を更新する

- README、使い方ガイド、必要であれば `--help` に、
  保存場所、上書き方法、削除方法を明記する
- Windows 利用者が「Credential Manager を直接触る必要があるのか」
  「アプリ側で完結できるのか」を迷わない説明にする

#### 記載したい内容の例

- 保存先: Windows Credential Manager
- 上書き方法: どのコマンドまたは手順を使うか
- 削除方法: どのコマンドまたは手順を使うか
- 削除後の挙動: 次回起動時に再入力が必要になること

### 要件5: テストを追加する

- 保存済みトークンありの通常起動
- 上書き後に新しい値が使われること
- 削除後に未設定扱いになること
- エラーメッセージとヘルプ文言が新しい導線に合っていること

## 検証方法

- `go test ./cmd/ow-custommatch-bot/...`
- Windows 実機でトークン再設定と削除を手動確認
- README / 使い方ガイドだけ読んで操作方法が理解できるか確認
