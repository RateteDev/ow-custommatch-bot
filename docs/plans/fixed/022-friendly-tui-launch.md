# 022-friendly-tui-launch

## Context

起動時のターミナル表示が技術者寄りで、非技術者には用途や状態が読み取りづらい問題がありました。
また、`.env` 前提の古い表示仕様が一部ドキュメントに残っており、実装と案内にずれがありました。

起動モードの文言、ヘッダー、保存先表示、起動ステップ表現を見直し、
初めて触る利用者でも状況を把握しやすい表示へ揃えます。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `cmd/ow-custommatch-bot/main.go` | 起動文言、バナー、パス表示、READY 表示の改善 |
| `cmd/ow-custommatch-bot/main_test.go` | 出力文言に関するテストを追加 |
| `docs/CLI表示仕様.md` | 現行実装に合わせて表示仕様を更新 |

## 実施順序

1. 起動モード文言を用途ベースに変更
2. バナーとパス表示を整理
3. 起動確定メッセージと READY 文言を更新
4. テストと仕様書を更新

## 要件

### 要件1: 起動モード文言の改善

- `本番モード` / `テストモード` を `通常運用` / `動作確認用` に変更する
- 各選択肢に用途説明を付与する

### 要件2: 起動導線の視認性向上

- バナーにガイド URL を表示する
- 保存先ラベルを日本語に変更する
- 起動確定文言を `PROD 通常運用で起動します。` / `TEST 動作確認用で起動します。` に揃える

### 要件3: READY 文言の更新

- READY 表示を `準備完了` ベースへ変更する

### 要件4: テストと仕様更新

- 出力文言のテストを追加する
- `.env` 前提の古い仕様記述を除去する

## 検証方法

- `go test ./cmd/ow-custommatch-bot/...`
- 起動時の表示を目視確認

## 実装結果

- `cmd/ow-custommatch-bot/main.go`
  - 起動モード文言を `通常運用` / `動作確認用` に変更
  - バナーにガイド URL を追加
  - パス表示ラベルを `データ保存先` / `ログファイル` / `データベース` に変更
  - 起動確定文言を `PROD 通常運用で起動します。` / `TEST 動作確認用で起動します。` に変更
  - `READY` 文言を `準備完了` ベースへ変更
- `cmd/ow-custommatch-bot/main_test.go`
  - 起動文言、バナー、パス表示、READY 文言のテストを追加
- `docs/CLI表示仕様.md`
  - `.env` 前提の古い記述を除去し、現行実装に合わせて更新

## 動作確認結果

- `go test ./...` 全件パス
- `go build ./...` 成功
- 起動時に `通常運用` / `動作確認用` の文言が自然に読めることを確認済み
- バナーにガイド URL が表示されることを確認済み
- パス表示が日本語ラベルで分かりやすいことを確認済み
- `準備完了` 表示が意図どおりに見えることを確認済み

## ユーザー確認コメント

- バナーの囲み枠内にツール名、バージョン、使い方 URL をまとめた方が見やすそう
- `データ保存先` は不要で、`ログファイル` と `データベース` のみでもよいのではないか
- Discord ログイン成功後に `/match` の案内まで表示した方が親切かもしれない

- READY 後の接続成功メッセージと利用案内の改善
