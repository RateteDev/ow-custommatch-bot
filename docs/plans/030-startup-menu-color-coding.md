# 030-startup-menu-color-coding

## Context

起動メニューは文言が整理されましたが、
3つの選択肢が同じ見た目のため、一覧性と視線誘導がやや弱い状態です。

利用者からも、番号と起動モードごとに色があった方が見やすいという要望が出ています。
既存の起動 UI には ANSI 色付けの仕組みがあるため、
過度に複雑な実装を追加せずに視認性を改善できます。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `cmd/ow-custommatch-bot/main.go` | 起動メニューの番号・見出し・補足文を色分け |
| `cmd/ow-custommatch-bot/main_test.go` | 非カラー出力の文言回帰と必要な表示テストを更新 |
| `docs/CLI表示仕様.md` | 色分け方針を反映 |

## 実施順序

1. 番号とラベルを同色で見せる方針に固定する
2. 既存の ANSI スタイル関数で配色を実装する
3. テストを更新する
4. 表示実装を調整する

## 要件

### 要件1: 選択肢を見分けやすくする

- `[1] 通常運用`
- `[2] 動作確認用`
- `[3] トークンを上書きする`

上記の番号とラベルを同じ色で表示する

#### 完了条件

- 3つの選択肢が視線で追いやすい
- 色がなくても文言だけで意味が分かる

### 要件2: 既存 UI と調和した配色にする

- バナー、ステップ表示、成功表示と極端に乖離しない
- 役割が異なる選択肢は色で区別できるようにする

- 通常運用は `green` で表示する
- 動作確認用は `yellow` で表示する
- トークン上書きは `cyan` で表示する

### 要件3: 非対応環境では自然にフォールバックする

- ANSI カラー非対応時は、従来どおりプレーンテキストで読めること
- 色に依存しない文言構造を維持すること

### 要件4: テストで文言を壊さない

- 非カラー出力の期待文言を維持する
- カラー有効時のエスケープシーケンスも局所的に確認する

## 検証方法

- `go test ./cmd/ow-custommatch-bot/...`
- カラー有効なターミナルで起動し、3つの選択肢が見分けやすいことを手動確認
- カラー無効相当の出力でも文言だけで意味が通ることを確認
