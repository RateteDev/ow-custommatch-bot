# 027-launch-ui-polish

## Context

起動 UI は 022 で大きく改善されましたが、ユーザー確認では
バナー内レイアウト、保存先表示項目、接続成功後の案内文に追加改善案が出ています。

現状の課題は次のとおりです。

- ツール名とバージョンは囲み枠内だが、URL は枠外にあり視線が散る
- `データ保存先`、`ログファイル`、`データベース` の3項目が本当に必要か判断が未整理
- `準備完了` は出るが、接続成功後に次の操作が分かる案内が弱い

細かな文言と視線誘導を見直し、起動後すぐに次の操作が分かる UI へ仕上げます。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `cmd/ow-custommatch-bot/main.go` | バナー、保存先表示、接続成功メッセージを改善 |
| `cmd/ow-custommatch-bot/main_test.go` | 表示文言テストを更新 |
| `docs/CLI表示仕様.md` | 最終表示仕様へ更新 |

## 実施順序

1. バナー内情報の配置を見直す
2. 保存先表示の項目数を整理する
3. 接続成功後の案内文を追加する
4. テストと仕様書を更新する

## 要件

### 要件1: バナー内に情報を集約する

- ツール名、バージョン、使い方 URL を囲み枠の中で見せる
- 1つの塊として見える配置にする

#### レイアウト例

```text
+-----------------------------------------------------------+
| OW CUSTOMMATCH BOT                                        |
| Version: v1.2.3                                           |
| 使い方: https://ratetedev.github.io/ow-custommatch-bot/   |
+-----------------------------------------------------------+
```

上記は一例ですが、少なくとも「URL だけが枠外にある状態」は解消することを目標にします。

ユーザーコメント：これでOK。完璧。

### 要件2: パス表示の項目を整理する

- `データ保存先` を残すか削るかを決め、表示項目を最適化する
- 残す場合も削る場合も、判断理由を説明できる状態にする

#### 検討対象

- `データ保存先`
- `ログファイル`
- `データベース`

#### 判断基準

- 非技術者が見て意味が分かるか
- トラブルシュート時に必要か
- 情報量が多すぎて起動時の視認性を落としていないか

ユーザーコメント：データ保存先は削除で。

### 要件3: 接続成功後の案内を追加する

- Discord ログイン成功後に、接続成功と次の操作が分かるメッセージを出す
- `/match` で募集開始できることをその場で伝える

#### 改善後のイメージ

```text
準備完了
Discord との接続に成功しました。
/match コマンドで募集を開始できます。
終了するには Ctrl+C を押してください。
```

ユーザーコメント：もうちょっと派手な方がいいかな。下のような感じでいい感じに色が欲しい。うろ覚えだからコマンド名は間違っているかも。
```text
🎉 Discord との接続に成功しました。
    "/match"コマンドで募集を開始できます。
    "/register"コマンドで~~。
    "/my_rank"コマンドで~~

    終了する場合は、ターミナルを閉じるか、Ctrl+Cを押してください。
```

#### 完了条件

- 「接続に成功したこと」が明確に伝わる
- 「次に何をすればよいか」がその場で分かる
- 既存の終了案内は残る

### 要件4: テストと仕様書を更新する

- バナー、パス表示、接続成功メッセージの期待文言をテストで固定する
- `docs/CLI表示仕様.md` を最終表示に合わせて更新する

## 検証方法

- `go test ./cmd/ow-custommatch-bot/...`
- 起動時の表示と接続成功後のメッセージを手動確認
