# 032-version-display-simplify

## Context

起動バナーでは `Version:` の後ろに `vv0.1.0-30-gf5b6d4c` のような値が出ることがあります。

これはビルド時に埋め込んだ `git describe` の結果に対し、
画面表示側でもさらに `v` を前置しているためです。
利用者向け表示としては情報量が多く、見た目も不自然です。

バナーや `--version` は、まず利用者が理解しやすい形式を優先し、
シンプルなバージョン表記へ統一します。

## 変更ファイル

| ファイル | 対応内容 |
|---------|---------|
| `cmd/ow-custommatch-bot/main.go` | バナーと `--version` の表示形式を整理 |
| `cmd/ow-custommatch-bot/main_test.go` | バージョン表示テストを更新 |
| `docs/CLI表示仕様.md` | バナーのバージョン表示例を更新 |

## 実施順序

1. 画面表示は `vX.Y.Z` 形式に統一する
2. 先に表示テストを追加/更新する
3. 余計な `v` 付与と詳細文字列露出を削る
4. 仕様書を更新する

## 要件

### 要件1: 利用者向け表示をシンプルにする

- 起動バナーの `Version:` は `v1.0.0` のような形式で表示する
- `vv1.0.0` のような二重 `v` を出さない

### 要件2: `--version` も同じ方針に揃える

- バナーと `--version` で極端に異なる見え方にしない
- 利用者がコピペしやすい形式にする
- `--version` は `ow-custommatch-bot v1.0.0` の形式で表示する

### 要件3: 開発用の詳細情報と利用者向け表示を分ける

- `-30-g<sha>` のような詳細表現は利用者向け表示に出さない
- 通常配布バイナリでは、利用者向け表示を常に簡潔な形式にする

### 要件4: テストで表示崩れを防ぐ

- バナーで `Version: v1.2.3` のように表示されること
- `--version` の期待形式を固定すること

## 検証方法

- `go test ./cmd/ow-custommatch-bot/...`
- `make build` 後に起動バナーの表示を確認
- `./bin/ow-custommatch-bot --version` の表示が簡潔になっていることを確認
