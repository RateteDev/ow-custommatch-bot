# CLI表示仕様

## 目的

起動時の CLI 表示を非技術者にも分かりやすくし、現在どこまで進んでいるか、どこにデータが保存されるか、次に何をすればよいかを迷わず把握できるようにします。

## 対象

- 実行ファイル起動時の CLI 表示
- 起動失敗時の案内メッセージ
- 実行中ログのコンソール表示
- ログファイル出力
- 引数による基本オプション

## CLIオプション

- `--help`
- `-h`
- `--version`
- `--test`

### `--help` の表示内容

- 使い方
- 利用可能なオプション一覧
- 初回起動時に `BOT_TOKEN` を入力すると保存されること
- Windows では Credential Manager に安全に保存すること
- 保存済みトークンを変えたい場合は起動メニューの `トークンを上書きする` を使うこと
- 詳細手順として `guideURL`（`https://ratetedev.github.io/ow-custommatch-bot/`）を案内すること

### `--version` の表示内容

- アプリケーション名
- バージョン文字列
- 形式は `ow-custommatch-bot v1.0.0` または `ow-custommatch-bot dev-abcdef0` とする
- 利用者向け表示では `git describe` 由来の `-30-g<sha>` のような詳細文字列を出さない

## 起動時のコンソール表示仕様

通常起動時は、起動処理の進行状況を段階的に表示します。

### バナー表示

- アプリ名
- バージョン
- 利用ガイド URL（`guideURL`）
- 上記 3 要素を同じ囲み枠の中に表示する
- `Version:` の表示例は `Version: v1.0.0` または `Version: dev-abcdef0` とする
- `Version: vv1.0.0` のような二重 `v` は表示しない

### 保存先表示

起動直後に次の 2 項目を日本語ラベル付きで表示します。

- `ログファイル`
  `.logs/` 配下に作成した今回のログファイルを表示します。
- `データベース`
  AppData 配下に保存する SQLite DB ファイルを表示します。

### 起動方法選択

対話可能なコンソールでは、用途ベースで起動方法を案内します。

- 見出しは「起動方法を選択してください」
- `[1] 通常運用`
  実際の運用として起動することを説明します。
- `[2] 動作確認用`
  テスト用ダミーデータで画面や流れを確認できることを説明します。
- `[3] トークンを上書きする`
  保存済み `BOT_TOKEN` を新しい値に更新できることを説明します。
- ANSI カラー有効時は `[1] 通常運用` を `green`、`[2] 動作確認用` を `yellow`、`[3] トークンを上書きする` を `cyan` で表示します。
- 色は番号とラベルの行に適用し、説明文は色に依存しない文面のまま表示します。
- `[3]` の説明に、保存先が Windows Credential Manager であることを表示します。
- `Enter` キーですぐ通常運用を開始できることを表示します。
- 無効入力時は `1 / 2 / 3 / Enter` のいずれかを再入力するよう案内します。
- `本番モード` `テストモード` という旧文言は表示しません。

### 起動確定メッセージ

- 通常運用時: `通常運用で起動します。`
- 動作確認用時: `動作確認用で起動します。`
- 起動メニューの `[1] 通常運用` / `[2] 動作確認用` と同じ語彙を使います。
- ANSI 色が無効な環境でも、文言だけでモードが判別できる状態を保ちます。

### 起動ステップ表示

- ステップは `1/3` のような進捗番号を表示します。
- 各ステップは `OK` / `FAIL` を表示します。
- 失敗時は失敗したステップで処理を終了します。

### 接続成功表示

- Discord ログイン成功後に成功メッセージを表示します。
- `/match`、`/register_rank`、`/my_rank` の導線を表示します。
- 終了方法として `Ctrl+C` を表示します。
- ANSI 色が無効でも文面だけで意味が通ることを保ちます。

## トークン保存仕様に基づく案内

- `BOT_TOKEN` は `.env` ではなく資格情報ストアから読み込みます。
- 初回起動時は対話入力で `BOT_TOKEN` を受け取り、保存します。
- Windows では Credential Manager を利用します。
- 保存済みトークンを更新したい場合は、起動メニューの `トークンを上書きする` から再入力します。
- 一般利用者向けに削除操作は表示しません。
- 未保存時や保存失敗時は `guideURL` を案内して自己解決しやすくします。

## エラー表示仕様（コンソール）

エラー発生時は、原因だけでなく対処先を含めて表示します。

### `BOT_TOKEN` が見つからない場合

- `BOT_TOKEN` が保存されていないことを明示します。
- 初回起動時に対話入力で保存できることを案内します。
- 詳しい手順として `guideURL` を表示します。

### 資格情報ストアが利用できない場合

- OS 都合で資格情報ストアを利用できないことを明示します。
- 対応手順として `guideURL` を表示します。

### トークン入力が空だった場合

- トークンが入力されなかったことを明示します。
- 詳しい手順として `guideURL` を表示します。

### Discord 認証に失敗した場合

- `websocket: close 4004: Authentication failed.` は通常の実行エラーと分けて扱います。
- いったん実行エラーを表示したうえで、`https://discord.com/developers/applications` を表示します。
- `Discord Developer Portal で BOT TOKEN を再発行または確認してください` と案内します。
- その場で新しい `BOT_TOKEN` の入力を受け付け、保存済みトークンを上書きします。
- 新しいトークン保存後の自動再試行は 1 回だけ行います。
- 再入力が空なら `トークンが入力されませんでした。` を含む案内で終了します。
- 保存失敗時は `トークンの保存に失敗しました` を含む案内で終了します。
- 再試行でも再度認証失敗した場合は、再試行失敗を明示して終了します。

### その他の起動エラー

- `起動に失敗しました` または `実行中にエラーが発生しました` を明示します。
- 利用者に見える範囲で必要な文脈を表示します。
- 必要に応じて `guideURL` を参照先として案内します。

## ログ出力仕様

### コンソール出力

- 起動時はバナー、保存先、起動方法選択、進捗表示、接続成功表示を優先します。
- 実行中ログはコンソールにも表示します。
- TTY 環境では色付き表示を行います。
- `NO_COLOR` 設定時は色を無効化します。

### ログファイル出力

- AppData 配下の `.logs/` に出力します。
- ファイル名はタイムスタンプ形式の `.log` です。
- ログファイルは装飾なしのプレーンログです。
- 起動時ログおよび実行中ログを記録します。

## 終了コード仕様

- `0`: 正常終了
- `1`: 起動失敗または実行中エラー
- `2`: 引数エラー

## 非対象

- Bot本体の動作仕様
- Discord上の機能仕様
- 内部実装詳細（関数分割、ログ実装方法、テスト構成など）
