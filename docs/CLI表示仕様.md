# CLI表示仕様

## 目的

実行ファイル実行時のCLI表示について、利用者が状態を把握しやすく、設定不備を自己解決しやすい表示仕様を定義します。

## 対象

- 実行ファイル起動時のCLI表示
- 起動失敗時の案内メッセージ
- 実行中ログのコンソール表示
- ログファイル出力
- 引数による基本オプション

## CLIオプション

- `--help`
- `-h`
- `--version`

### `--help` の表示内容

- 使い方
- 利用可能なオプション一覧
- 起動前に必要な準備（`.env` 配置、`BOT_TOKEN` 設定）

### `--version` の表示内容

- アプリケーション名
- バージョン文字列

## 起動時のコンソール表示仕様

通常起動時は、起動処理の進行状況を段階的に表示します。

- バナー（アプリ名、バージョン）
- 実行ディレクトリ
- ログファイル保存先
- DBファイルパス
- 起動ステップの進行状況（例: `1/4` 形式）
- 接続開始の案内
- 終了操作の案内（`Ctrl+C`）

### 起動ステップ表示

- ステップは `OK` / `FAIL` を表示
- 失敗時は失敗したステップで処理を終了

## エラー表示仕様（コンソール）

エラー発生時は、原因だけでなく対処方法を含めて表示します。

### `.env` が見つからない場合

- `.env` の不足を明示
- 配置場所（実行ファイルと同じフォルダ）を案内
- 設定例（`BOT_TOKEN=YOUR_DISCORD_BOT_TOKEN`）を表示
- 詳細手順として同じフォルダの `使い方.html` を案内

### `BOT_TOKEN` が未設定の場合

- 必須設定不足であることを明示
- `.env` に設定する例（`BOT_TOKEN=YOUR_DISCORD_BOT_TOKEN`）を表示
- `BOT_TOKEN=YOUR_DISCORD_BOT_TOKEN` のままでも未設定扱いであることを明示
- 詳細手順として同じフォルダの `使い方.html` を案内

### `.env` の書式が不正な場合

- `.env` の書式不正を明示
- `KEY=VALUE` 形式であることを案内
- 設定例（`BOT_TOKEN=YOUR_DISCORD_BOT_TOKEN`）を表示
- 詳細手順として同じフォルダの `使い方.html` を案内

### その他の起動エラー

- エラー種別を明示（起動失敗 / 実行中エラー）
- 利用者に見える範囲で必要な文脈を表示（例: 対象ファイルパス）
- 必要に応じて `使い方.html` の参照を案内

## ログ出力仕様

### コンソール出力

- 起動時は整形表示（バナー、ステップ表示、案内文）を優先
- 実行中ログはコンソールにも表示される
- TTY 環境では色付き表示を行う
- `NO_COLOR` 設定時は色を無効化する

### ログファイル出力

- 実行ファイルと同じフォルダ配下の `.logs/` に出力する
- ファイル名はタイムスタンプ形式（`.log`）
- ログファイルは装飾なしのプレーンログ（時刻付き）
- 起動時ログおよび実行中ログを記録する

## 終了コード仕様

- `0`: 正常終了
- `1`: 起動失敗または実行中エラー
- `2`: 引数エラー

## 非対象

- Bot本体の動作仕様
- Discord上の機能仕様
- 内部実装詳細（関数分割、ログ実装方法、テスト構成など）
