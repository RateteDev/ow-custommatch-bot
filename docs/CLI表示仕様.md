# CLI表示仕様

## 目的

起動時の CLI 表示を非技術者にも分かりやすくし、現在どこまで進んでいるか、どこにデータが保存されるか、困ったときに何を見ればよいかを迷わず把握できるようにします。

## 対象

- 実行ファイル起動時の CLI 表示
- 起動失敗時の案内メッセージ
- 実行中ログのコンソール表示
- ログファイル出力
- 引数による基本オプション

## CLIオプション

- `--help`
- `-h`
- `--version`
- `--test`

### `--help` の表示内容

- 使い方
- 利用可能なオプション一覧
- 初回起動時に `BOT_TOKEN` を入力すると保存されること
- Windows では Credential Manager に安全に保存すること
- 詳細手順として `guideURL`（`https://ratetedev.github.io/ow-custommatch-bot/`）を案内すること

### `--version` の表示内容

- アプリケーション名
- バージョン文字列

## 起動時のコンソール表示仕様

通常起動時は、起動処理の進行状況を段階的に表示します。

### バナー表示

- アプリ名
- バージョン
- 利用ガイド URL（`guideURL`）

### 保存先表示

起動直後に次の 3 項目を日本語ラベル付きで表示します。

- `データ保存先`
  AppData 配下のアプリ用ディレクトリを表示します。
- `ログファイル`
  `.logs/` 配下に作成した今回のログファイルを表示します。
- `データベース`
  AppData 配下に保存する SQLite DB ファイルを表示します。

### 起動方法選択

対話可能なコンソールでは、用途ベースで起動方法を案内します。

- 見出しは「起動方法を選択してください」
- `[1] 通常運用`
  実際の運用として起動することを説明します。
- `[2] 動作確認用`
  テスト用ダミーデータで画面や流れを確認できることを説明します。
- `Enter` キーですぐ通常運用を開始できることを表示します。
- 未入力時は一定時間後に通常運用で自動起動することを表示します。
- `本番モード` `テストモード` という旧文言は表示しません。

### 起動確定メッセージ

- 通常運用時: `PROD 通常運用で起動します。`
- 動作確認用時: `TEST 動作確認用で起動します。`

### 起動ステップ表示

- ステップは `1/3` のような進捗番号を表示します。
- 各ステップは `OK` / `FAIL` を表示します。
- 失敗時は失敗したステップで処理を終了します。

### READY表示

- 起動準備完了時は `準備完了` を表示します。
- 続けて `Discordへの接続を開始します。終了するには Ctrl+C を押してください。` を表示します。

## トークン保存仕様に基づく案内

- `BOT_TOKEN` は `.env` ではなく資格情報ストアから読み込みます。
- 初回起動時は対話入力で `BOT_TOKEN` を受け取り、保存します。
- Windows では Credential Manager を利用します。
- 未保存時や保存失敗時は `guideURL` を案内して自己解決しやすくします。

## エラー表示仕様（コンソール）

エラー発生時は、原因だけでなく対処先を含めて表示します。

### `BOT_TOKEN` が見つからない場合

- `BOT_TOKEN` が保存されていないことを明示します。
- 初回起動時に対話入力で保存できることを案内します。
- 詳しい手順として `guideURL` を表示します。

### 資格情報ストアが利用できない場合

- OS 都合で資格情報ストアを利用できないことを明示します。
- 対応手順として `guideURL` を表示します。

### トークン入力が空だった場合

- トークンが入力されなかったことを明示します。
- 詳しい手順として `guideURL` を表示します。

### その他の起動エラー

- `起動に失敗しました` または `実行中にエラーが発生しました` を明示します。
- 利用者に見える範囲で必要な文脈を表示します。
- 必要に応じて `guideURL` を参照先として案内します。

## ログ出力仕様

### コンソール出力

- 起動時はバナー、保存先、起動方法選択、進捗表示、準備完了表示を優先します。
- 実行中ログはコンソールにも表示します。
- TTY 環境では色付き表示を行います。
- `NO_COLOR` 設定時は色を無効化します。

### ログファイル出力

- AppData 配下の `.logs/` に出力します。
- ファイル名はタイムスタンプ形式の `.log` です。
- ログファイルは装飾なしのプレーンログです。
- 起動時ログおよび実行中ログを記録します。

## 終了コード仕様

- `0`: 正常終了
- `1`: 起動失敗または実行中エラー
- `2`: 引数エラー

## 非対象

- Bot本体の動作仕様
- Discord上の機能仕様
- 内部実装詳細（関数分割、ログ実装方法、テスト構成など）
